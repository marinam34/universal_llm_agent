# Universal Website Agent Prototype

Прототип создаёт универсального внешнего агента по одному URL сайта: анализирует интерфейс, создаёт агента для конкретного домена, принимает команды в чате и исполняет безопасные шаги через Playwright с human-in-the-loop перед финальным необратимым действием.

## Что умеет

- Даёт стартовый экран с одним полем URL и кнопкой `Создать агента`.
- После обхода сайта открывает рабочий экран: сайт в основном окне и чат-виджет агента справа снизу.
- Обходит сайт в пределах домена и сохраняет:
  - страницы и URL,
  - ссылки, кнопки, формы,
  - поля ввода и простые подписи,
  - граф переходов.
- Строит план действий для пользовательской задачи:
  - определяет, возможно ли выполнить задачу,
  - перечисляет недостающие данные,
  - возвращает текстовое объяснение и строгий JSON-план.
- Исполняет план через Playwright только безопасными действиями:
  - `goto(url)`
  - `click(selector)`
  - `type(selector, value)`
  - `select(selector, option)`
  - `wait_for(selector/text)`
- Не выполняет автоматически:
  - submit,
  - confirm,
  - pay,
  - delete.
- После каждого шага делает простую валидацию:
  - проверка URL,
  - проверка ожидаемого элемента/текста,
  - поиск признаков ошибки на странице.
- Сохраняет артефакты:
  - `knowledge_base.json`
  - `action_plan.json`
  - `execution_log.json`

## Архитектура

```text
.
├── agent/      # LLM-адаптер и построение плана
├── backend/    # FastAPI API и раздача frontend
├── core/       # shared config и pydantic-модели
├── crawler/    # обход сайта и сбор интерфейса
├── executor/   # безопасное исполнение шагов через Playwright
├── frontend/   # HTML/CSS/JS интерфейс
├── kb/         # файловое хранение JSON-артефактов
├── artifacts/  # создаётся автоматически
├── .env.example
├── pyproject.toml
└── README.md
```

## Запуск

### 1. Установка

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -e .
playwright install chromium
cp .env.example .env
```

Если используете conda:

```bash
conda activate agent
pip install -e .
playwright install chromium
```

### 2. Настройка LLM

Без LLM ключа проект всё равно работает, но использует эвристический fallback-планировщик.

Для OpenRouter укажите в `.env`:

```env
OPENROUTER_API_KEY=your_key
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=openai/gpt-4o-mini
```

### 3. Старт

```bash
uvicorn backend.main:app --reload
```

Откройте [http://127.0.0.1:8000](http://127.0.0.1:8000).

## Как использовать

1. Введите URL сайта на стартовом экране.
2. Нажмите `Создать агента`.
3. Дождитесь завершения обхода сайта.
4. После открытия рабочего экрана пишите команды прямо в чат, например: `Запиши меня к врачу на среду после 18:00`.
5. Если агенту не хватает данных, он попросит уточнения.
6. При необходимости заполните блок `Данные пользователя (JSON)`.
7. Агент построит план, покажет его JSON и сразу выполнит безопасные шаги.
8. Перед финальным необратимым действием агент остановится и попросит завершить действие вручную.

## API

### `POST /api/create-agent`

```json
{
  "url": "https://example.com",
  "max_pages": 15
}
```

Создаёт KB и возвращает метаданные агента для интерфейса.

### `POST /api/chat`

```json
{
  "url": "https://example.com",
  "message": "Запиши меня на анализ крови",
  "user_inputs": {
    "full_name": "Иван Иванов",
    "phone": "+79990000000",
    "date": "2026-03-05",
    "time": "10:00"
  }
}
```

Строит план и, если данных достаточно, сразу запускает безопасное исполнение.

### `POST /api/crawl`

Низкоуровневый endpoint для ручного запуска обхода.

### `POST /api/plan`

Низкоуровневый endpoint для ручной генерации JSON-плана.

### `POST /api/execute`

Низкоуровневый endpoint для исполнения уже готового плана.

## Ограничения прототипа

- iframe превью сайта зависит от `X-Frame-Options` и CSP конкретного сайта; если сайт запрещает встраивание, используйте отдельное окно Playwright.
- Выбор селекторов сейчас эвристический; для сложных SPA-сценариев нужна доработка и более устойчивые локаторы.
- Планировщик без LLM работает на упрощённых правилах.
- Не реализованы логин-сессии, anti-bot обход и капча.
